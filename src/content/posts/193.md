共计两个步骤, 使用`ipaddress.com`测试节点来检测当前网络Github最佳目标IP

### 1.获取`github.com`域名解析IP
<a href="https://www.ipaddress.com/site/github.com#ipinfo">点击获取github.com域IP</a>

找到如图位置，复制ip得到`140.82.113.3`（注意每个人从不通网络运营商下得到的不一定一致）
![a.png][1]

### 2.获取`github.global.ssl.Fastly.net`域名解析IP
<a href="https://www.ipaddress.com/site/github.global.ssl.fastly.net#ipinfo">点击获取github.global.ssl.Fastly.net域IP</a>
找到如图位置，可以看到有四个IP，随意复制ip得到`140.82.113.3`（注意每个人从不通网络运营商下得到的不一定一致）
![b.png][2]

### 3.修改`Hosts`文件（两个域名对应的IP一定要自己去获取）
#### a. Linux/Mac
```
# 需要拥有root权限(已拥有跳过此步骤)
```
sudo passwd
# 执行后需要输入密码，回车后再输入确认密码（两次密码保持一致回车）
```
# 修改
sudo
echo '140.82.113.3        github.com' >> /etc/hosts
echo '151.101.1.194       github.global.ssl.fastly.net' >> /etc/hosts
```
#### b. Windows
直接修改`C:\Windows\System32\drivers\etc\hosts`文件， 
添加：
```
140.82.113.3        github.com
151.101.1.194       github.global.ssl.fastly.net
```
